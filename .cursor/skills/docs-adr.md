# Skill: Docs and ADR

Use this skill when a code change changes architecture, data model, protocol, or reliability behavior.

## ADR Template

```md
# ADR NNNN: Title

## Background

## Options

## Decision

## Consequences

## Follow-up Tasks
```

## Documentation Targets

- REST changes: `docs/api.md`.
- DB changes: `docs/data-model.md`.
- WS changes: `docs/websocket-protocol.md`.
- Reliability changes: `docs/reliability.md`.
- Scaling/cache/MQ changes: `docs/scaling.md`.
- Long-term extension changes: `docs/extensions-roadmap.md`.
