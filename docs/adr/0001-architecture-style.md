# ADR 0001：采用模块化单体作为初始架构

## 背景

EchoLine 的目标是训练真实消息平台的工程能力，但项目初期不应因为微服务治理、部署和跨服务调试消耗过多精力。

## 决策

初始阶段采用模块化单体：

- 单个 API 服务承载核心同步链路。
- 使用清晰的 internal module 边界。
- 使用 worker 进程承载异步任务。
- 通过 event bus 接口为 Kafka/Redpanda 留出替换空间。

## 理由

- 更容易本地开发和测试。
- 更容易让 Cursor Cloud Agent 持续推进。
- 能先把业务模型、可靠性和系统设计问题做扎实。
- 后续可以按模块边界拆分服务。

## 代价

- 初期无法真实体现跨服务部署复杂度。
- 需要保持模块边界，避免变成无边界大泥球。
